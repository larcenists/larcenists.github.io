Bug Reports

Bug reports may be submitted at Larceny's GitHub site or by email to larceny@ccs.neu.edu.

Lists of all submitted bugs are also available.

Known Bugs in Larceny

Some of the more important bugs that have been discovered in Larceny v0.98 "General Ripper" include:

cond-expand doesn't work at expression level not yet fixed
arbitrary ellipsis feature of R7RS syntax-rules is not yet implemented not yet fixed
SRFI 42 doesn't work in R7RS read/eval/print loop not yet fixed
eval allows assignments to imported primitives not yet fixed
R7RS/R6RS libraries that are defined in the same file with a program that imports them can be invoked more than once not yet fixed
compile-stale-libraries often recompiles files unnecessarily not yet fixed
SRFI 114 hash functions don't work on empty strings/vectors fixed in development version

Bugs fixed in Larceny v0.98

Some of the more important bugs in v0.97 that have been fixed in v0.98 include:

+i reads as 0-1i fixed in v0.98b1
ERR5RS/R6RS (or) returns #t fixed in v0.98b1

Enhancements added in Larceny v0.98

Some of the more important improvements added in v0.98 include:

R7RS programs and define-library now work done in v0.98
R7RS syntax and procedures are now implemented done in v0.98
circular structures are now written using R7RS syntax done in v0.98
Larceny has been upgraded to Unicode 7.0.0 done in v0.98
Larceny now supports SRFI 101, 112, 113, 114, 115, and 116 done in v0.98
